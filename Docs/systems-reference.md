# Systems Reference — mechanics, builders & content authoring

**Written 2026-07-12 (W5.11), current through W5.10 (suite 969).** Open this when working on any interaction mechanic, running builder menus, or authoring content. Companion: `Docs/gameplay-flow.md` (what the player experiences), `Docs/experiments-reference.md` (per-experiment data). Code convention everywhere: **thin MonoBehaviours over pure, suite-tested C# cores**, every component with a `Bind()` seam (edit-mode `AddComponent` fires no `Awake`/`OnEnable`).

## 1. Liquids
- **`LiquidPhysics`** (on every bottle/vessel; needs a Renderer host): `maxVolume` 1000, default volume **0** (the wake-from-empty contract). `AddLiquid`: capacity guard first (overflow → `LiquidRejected`, never completes tasks) → `LiquidAdded` fires → EMPTY vessel adopts the incoming chemical (+visuals); same chemical tops up; different chemical + registry → `FindReaction` hit = swap to `resultLiquid` (+precipitate, colour lerp, SFX, `ReactionOccurred`) / miss = volume adds + `WrongReagentMixed`. `SetContents(chem, ml)` = builder/test seam (edit-safe, resets the **`Ledger`** — a pure `VesselLedger` "Ethanol 120 ml + NaOH 50 ml" story used by hover/popups). `PourOut` leaves `currentChemical` at 0 ml (display shows "empty" via `IsEmpty`). Visual contract: `mainRenderer` must carry the `PharmaSynth/Liquid` shader (`_Fill/_LiquidColour/...`); play-mode-only material work.
- **`LiquidPourer`**: tilt >45° + volume>0 → `PourTick`: `RaycastAll` straight down from the mouth Spout, **triggers ignored**, own hierarchy skipped (`ResolveTarget` — self-tilt can never fill/complete anything); W5.12 pour assist: a missed ray falls back to a `SphereCastAll` (r 0.045, `assistRadius`) so a receiver the stream narrowly misses still catches — the visual arc lands where the transfer actually goes. Nearest other `LiquidPhysics` receives, else waste + puddle (+acid hazard). 35 ml/s max. Visible stream (particles + arc line) + looping 3D pour audio. Dev key **P** = pour-debug overlay (floating "hit/target" text per pouring mouth). `PowderPourer` mirrors it for powders.
- **Receiving vessels** are built visible-ready: `BuildVessel` swaps empty prefabs → `_WithLiquid` twins and ALWAYS `SetContents` explicitly; `EnsureLiquidVisual` (public, also used by `ShelfPourWiring`) adds an inset liquid mesh when a prefab lacks one.
- **`ReagentSupplyMonitor`**: polls needed-vs-available for pour steps; latches → SupplyPrompt; `Unlatch(20s)` on "keep trying"; demo = auto-refill.

## 2. Tool verbs (W5.8)
- **Stir** — `StirController` on the vessel + pure `OrbitMath` (swept-angle accumulation, ±180° wrap-safe, 45°/sample anti-teleport clamp, leaving PAUSES never resets). Needs >1 ml in the vessel; works held or on table. Progress popups at 25% steps.
- **Grind** — `GrindController` on the mortar (pestle orbits the bowl; done → powder heap + puff). Null taskId = educational-only (staged mortars everywhere grind). Methane `prepare-mixture` = dual-path (grind OR legacy zone-touch).
- **Weigh** — `WeighStation` (pan trigger over the Balance fixture) + pure `WeighMath` (contents grams at 1 g/ml proxy, auto-tare; `PanSettled` ≥0.75 s; chemical-mode = vessel holds ≥90% of required chemical, item-mode = named tool on pan). Drives `WeighingScaleController`'s live "0.00 g" display. Layout `sim: 7`; the matching pour binding uses `completesTask: 0`.
- **Matches & burners** — `Matchstick.ShouldStrike` (held swipe ≥0.35 m/s on a `MatchStrikerSurface`: matches box, burner bases, staged striker) or secondary self-light near an already-hot sim. `BurnerController.Ignite` (lit match ≤0.18 m) = flame + light; **Heat stations whose required prop is a burner only heat while lit** (`ZoneSimStation.SetIgnitionGate`; billboard hints "Light the burner"). Methane splint: lit match ≤0.25 m of the filled tube → pop (+20 s auto-fallback).
- **Scoop (W5.12)** — `ScoopController` on Scoopula/Spatula + pure `ScoopMath`: while HELD, dipping into a Solid/Powder store picks up a fixed 2 g charge (tinted heap rides the blade, `+2 g` popup); touching another vessel deposits it (`AddLiquid` → balance/status update; running-total popup). Sources = any `LiquidPhysics` whose chemical `state` is Solid/Powder (KMnO₄ + NaNO₂ re-marked Solid in W5.12; `Tools ▸ PharmaSynth ▸ Audit Chemical States` reports the table).
- **Clean (W5.12)** — `CleanableVessel` (+pure `CleanupMath`) on every receiver: emptying a used vessel leaves residue (label reads "Dirty …"); the `TestTubeBrush` (`BrushController`) scrubs — each 0.12 m of held brush travel = one swipe, −20% with a popup, 5 swipes → "Clean …" label; wash-bottle water rinses too (1.2%/ml). Educational only, never graded.
- All verb controllers re-`Register()` their TaskGraph conditions on `ExperimentStarted` AND on re-enable (retry-safe; `RegisterCondition` ignores unknown ids).
- **Apparatus assembly (W5.12)** — `ApparatusSnap` + pure `AssemblyMath`: releasing a part near a compatible host SNAPS it on (directed pairs: gauze/clay-triangle→tripod, crucible→clay triangle, vessels→gauze, iron ring→retort-stand pole, burners→under-tripod, watch glass→beaker mouth; seats = TopCenter/SameBase/PoleMid). Attached parts parent to the host, go kinematic, forward their colliders to the ROOT's grab → **grabbing any member moves the whole assembly, formation preserved**; the **ACTIVATE click** (trigger) pops the newest part off (deepest chain first) — grab never dismantles. `DropRespawn.Suspended` while attached. Wired by `WireSceneItem`/kits builder for participating prefabs (station sockets untouched).

## 3. Breakage & physics
- **`Mishandling.Breakables`** = THIN GLASS ONLY (beakers, flasks, graduated cylinders, test tubes, vials, watch glass + `_WithLiquid` twins). GlassRod/Funnel/Dropper/EvaporatingDish/Crucible/Mortar/Pestle are DELISTED (still clink via the ceramic set); Balance/burners/ClayTriangle = metal set. `DefaultBreakSpeed = 7.0` (~2.5 m fall — W5.12 leniency: bench-height drops always survive; only extreme falls/throws shatter). Impact clink volume scales with speed (`ImpactVolume01` — soft set-downs whisper). `BreakableGlassware` guards: kinematic-immune, held-immune (`isSelected`), 0.35 s release grace, 1.5 s re-arm; `Break()` = mistake + shatter + spill puddle + `DropRespawn.GoHome()` (fresh replacement). Mistake/shatter messages use `Mishandling.DisplayNameFor` (LabItem → chemical → prettified name — never GO code names). `Physics Audit (Fix Scene Items)` strips stale BreakableGlassware from non-glass, re-keys ImpactSound by nature, re-pins serialized thresholds.
- **`DropRespawn`**: kill-Z instant respawn; idle ≥25 s AND >0.4 m from home AND **resting below y=0.5 (floor only — W5.12)** → quiet return; items staged on tables/shelves are NEVER reclaimed mid-run. `GoHome()` **restores the captured home supply** (`CaptureSupply` on first play Update) — an exhausted bottle respawns FULL and visible (lab tour = unlimited by nature; campaign starvation still reachable while the empty bottle stays on the bench). **settle-freeze** = released body at rest 2.5 s → kinematic in place. `Suspended` = assembly-managed (no respawn while attached). `SetHome` is the builder-owned seam; **Tools ▸ PharmaSynth ▸ Re-Home Scene Items (Adopt Current)** adopts current transforms as homes after ANY manual re-arrangement.
- **`PhysicsProfiles.EnsurePhysics`** (mass + convex colliders — dynamic bodies need convex MeshColliders or they fall through the world) + `RealSizes` (real-world scale by longest axis; never height-normalise flat tools) + `GrabTuning` (velocity-tracked grabs; held items collide) + `GrabPhysicsPolicy` (kinematic-parked ↔ dynamic on release).

## 8. Content authoring (add/change content without engine work)
1. **Chemical**: create `Chem_*.asset` (ChemicalData) under ScriptableObjects/Chemicals + register in `SceneAssetLibrary.chemicals` (or add a `RawReagentCatalog` row and run `Generate Raw Reagent Data`). Names are looked up by exact `chemicalName` string.
2. **Reaction**: `ReactionRule` asset (inputs = ChemicalData ASSETS, order-independent; result/precipitate/outcome/`expectedObservation`) + add to `MasterReactionRegistry.rules`. ⚠ If a layout pours reagent X for a test, the RULE inputs must reference the same chemical asset — swap both together.
3. **Experiment**: `ExperimentModuleDefinition` (graphTasks: taskId/label/phase/prereqs/skill/rubric/hint; trackedSkills; masteryThreshold; parTime) + `Layout_*.asset` (stations/props/vessels+bindings — see experiments-reference.md for the schema in action) + `Quiz_*` (3 MCQs) + 4 cutscenes (intro/prep/success/failure) wired in `CutsceneLibrary` + a `CatalogEntry` in `ExperimentCatalog` (the one code touch) + `ExperimentLibrary`/`QuizBankLibrary` registration. Then the layout/verb chain (§7) + suite (ContentSuite pins task counts — update the pin).
4. **Vessel test bindings** must land on the SEEDED test vessel (the W5.9 applier's `EnsureBinding` prefers same-task → seeded → any).
5. **Dialogue**: pools in `PharmeeLines` (suite-pinned counts) — remember the voice-hash caveat (§5).
6. **Suite discipline**: every new mechanic ships a pure core + suite assertions; pinned behavior assertions move IN THE SAME COMMIT as the behavior change; suite must stay green after every phase of work.
